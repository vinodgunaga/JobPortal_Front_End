## 📂 Directory Purposes

### `/src/components`
Reusable React components organized by feature:

- **`auth/`** - Authentication-specific components (ProtectedRoute)
- **`jobs/`** - Job-specific components (JobCard, JobFilters)
- **`layout/`** - App structure components (Navbar, Layout)
- **`ui/`** - Generic, reusable UI components (Button, Input, Card)

**Convention:** 
- Each component in its own file
- PascalCase naming (e.g., `JobCard.tsx`)
- Export component as named export

### `/src/pages`
Page-level components that represent routes:

- **`auth/`** - Login and registration pages
- **`jobs/`** - Job browsing and detail pages
- **`recruiter/`** - Recruiter-specific pages

**Convention:**
- Pages end with `Page.tsx` (e.g., `LoginPage.tsx`)
- One route per file
- Exported as named export

### `/src/services`
API integration layer:

- **`api.ts`** - Axios instance with interceptors, error handling
- **`auth.service.ts`** - Authentication API calls
- **`job.service.ts`** - Job-related API calls

**Convention:**
- Export object with methods (e.g., `authService.login()`)
- All async functions return Promises
- TypeScript generics for responses

### `/src/store`
Redux state management:

- **`slices/`** - Redux Toolkit slices (state + reducers + actions)
- **`hooks.ts`** - Typed `useAppDispatch` and `useAppSelector`
- **`store.ts`** - Store configuration

**Convention:**
- One slice per domain (auth, jobs, etc.)
- Async logic in `createAsyncThunk`
- Export actions and reducer

### `/src/types`
TypeScript type definitions:

- **`auth.types.ts`** - Auth-related interfaces and enums
- **`job.types.ts`** - Job-related interfaces and enums

**Convention:**
- Interfaces for objects
- Enums for fixed sets of values
- Match backend API response shapes

## 🔄 Data Flow

```
User Interaction
    ↓
Component (Page/UI Component)
    ↓
Dispatch Action (useAppDispatch)
    ↓
Redux Slice (Async Thunk)
    ↓
Service Layer (API call)
    ↓
Axios Instance (with interceptors)
    ↓
Backend API
    ↓
Response
    ↓
Redux State Updated
    ↓
Component Re-renders (useAppSelector)
    ↓
UI Updates
```

## 📝 Naming Conventions

### Files
- **Components:** PascalCase (e.g., `JobCard.tsx`)
- **Types:** camelCase with `.types.ts` suffix (e.g., `job.types.ts`)
- **Services:** camelCase with `.service.ts` suffix (e.g., `auth.service.ts`)
- **Styles:** kebab-case (e.g., `global-styles.css`)

### Code
- **Components:** PascalCase (e.g., `const JobCard = () => {}`)
- **Functions:** camelCase (e.g., `const fetchJobs = () => {}`)
- **Constants:** UPPER_SNAKE_CASE (e.g., `const API_BASE_URL`)
- **Interfaces:** PascalCase with descriptive names (e.g., `interface JobCardProps`)
- **Enums:** PascalCase (e.g., `enum JobType`)

## 🎨 Component Patterns

### Presentational Component
```typescript
// components/ui/Button.tsx
interface ButtonProps {
  variant?: 'primary' | 'secondary';
  children: React.ReactNode;
}

export const Button: React.FC<ButtonProps> = ({ variant, children }) => {
  return <button className={...}>{children}</button>;
};
```

### Container Component
```typescript
// pages/jobs/JobListPage.tsx
export const JobListPage: React.FC = () => {
  const dispatch = useAppDispatch();
  const { jobs } = useAppSelector(state => state.jobs);
  
  useEffect(() => {
    dispatch(fetchJobs());
  }, []);
  
  return <div>...</div>;
};
```

### Form Component
```typescript
// pages/auth/LoginPage.tsx
export const LoginPage: React.FC = () => {
  const { register, handleSubmit } = useForm<LoginRequest>();
  
  const onSubmit = (data: LoginRequest) => {
    // Handle form submission
  };
  
  return <form onSubmit={handleSubmit(onSubmit)}>...</form>;
};
```

## 📦 Import Aliases

Use `@/` prefix for cleaner imports:

```typescript
// Instead of
import { Button } from '../../../components/ui/Button';

// Use
import { Button } from '@/components/ui/Button';
```

Configured in:
- `tsconfig.json` (for TypeScript)
- `vite.config.ts` (for Vite)

## 🎯 State Management Pattern

```typescript
// 1. Define types
interface JobState {
  jobs: Job[];
  loading: boolean;
}

// 2. Create slice
export const jobSlice = createSlice({
  name: 'jobs',
  initialState,
  reducers: {},
  extraReducers: (builder) => {
    builder.addCase(fetchJobs.fulfilled, (state, action) => {
      state.jobs = action.payload;
    });
  },
});

// 3. Use in components
const { jobs, loading } = useAppSelector(state => state.jobs);
```

## 🌐 API Service Pattern

```typescript
export const jobService = {
  async getJobs(params: JobQueryParams): Promise<PagedResult<Job>> {
    return apiClient.get<PagedResult<Job>>('/jobs', { params });
  },
  
  async createJob(data: CreateJobRequest): Promise<Job> {
    return apiClient.post<Job>('/jobs', data);
  },
};
```
